수치 불일치 발생 시 원인을 역추적하고 근거를 문서화하는 업무를 수행했습니다. QA의 버그 리포팅 및 재현 과정과 같은 사고방식입니다.

비즈니스 흐름 전체를 파악하며 어느 지점에서 리스크가 발생하는지 판단하는 역량은, 테스트 범위와 우선순위를 결정하는 QA 관점과 맞닿아 있습니다.

Python · Pytest · Selenium · Playwright 전 과정 실습

완료
기술 스택PythonPytestSeleniumPlaywright
▶ Python 기본 문법 실습
  • 자료구조 (리스트, 튜플, 딕셔너리) 생성·수정·삭제
  • 조건문, 반복문 (for / while / break / continue)
  • 함수 정의, 기본값 인자, 가변 인자, lambda

▶ Pytest 단위 테스트 실습
  • mycalc / Calculator 클래스 — add, subtract, divide 테스트
  • pytest.raises(), pytest.approx() 활용
  • fixture (scope: function / class / module), yield fixture
  • parametrize 기본 · 중첩 · CSV 파일 기반 데이터 테스트
  • conftest.py 공통 fixture 관리, pytest_generate_tests 활용
  • member_manager 회원관리 시스템 테스트 설계 및 구현
    → 회원 추가/조회/삭제 · 정상/비정상/경계값 케이스 · fixture 상태(없음/1명/여러명) 분리
▶ Selenium UI 자동화 실습
  • WebDriver 실행, 페이지 이동, 요소 탐색 (By.ID / CSS_SELECTOR 등)
  • WebDriverWait + expected_conditions 적용
  • Page Object Model (LoginPage, SignupPage) 구조 설계
  • 로그인 성공/실패 시나리오 자동화 (parametrize 적용)
  • 회원가입 입력 검증 자동화
  • headless 옵션, conftest driver fixture 분리
▶ Playwright UI 자동화 실습
  • browser / context / page 구조 이해 및 멀티 페이지 실습
  • POM 구조 설계 (LoginPage, SignupPage — Playwright 버전)
  • 동적 로딩 테스트 (Selenium vs Playwright 비교 구현)
  • expect 기반 assertion (to_contain_text, to_have_url, to_have_text)
  • Mock API 응답 처리 (page.route 활용, 성공/실패 케이스)
  • conftest browser / page fixture (session scope) 분리
